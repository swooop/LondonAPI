Life is hard

Thankfully to we are now down to one HTTP request

						
				curl --user "$MJ_APIKEY_PUBLIC:$MJ_APIKEY_PRIVATE" \
        https://api.mailjet.com/v3/send \
        -F from='you@example.com' \
        -F to='receiver@example.com' \
        -F subject='Hello Paris' \
        -F text='Greetings from the city of lights.'

What do we do to aid deliverability?

  • SPF and DKIM
  • Dedicated IPs
  • Realtime statistics, feedback loops and good relationships with ISP's
  • Highly optimised send rates
  • A newsletter tool.

So what about the API?

  • It's written in Pascal. Mostly.
    • Type safe
    • Pretty quick compilers
    • Good with strings (null terminated)
  • Very easy to integrate with C/C++
  • It's pretty RESTful

So, much easy email management:

Create a contact list... 


						
						curl -X POST --user "$MJ_APIKEY_PUBLIC:$MJ_APIKEY_PRIVATE"
https://api.mailjet.com/v3/REST/contactslist -H
"Content-Type: application/json" -d
'{"Name":"January Newsletter List"}'

Add to a contact list:


						
						curl -X POST --user "$MJ_APIKEY_PUBLIC:$MJ_APIKEY_PRIVATE"
https://api.mailjet.com/v3/REST/listrecipient -H "Content-Type: application/json" -d
"{'ContactID':1, 'ListID':2, 'IsActive':True}"
